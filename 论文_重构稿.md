## 摘要

随着计算机科学的发展,图论与图算法在网络分析、路径规划、社交网络挖掘等领域应用日益广泛,而对于图算法的学习与教学,离不开对其算法过程与图结构的直观理解。图论是计算机科学中的一个重要分支,主要关注顶点与边的组织方式以及在其上的算法设计,为了能够更好的了解其基本内容和其运行原理,设计和开发了一款简单而能够基本实现图算法可视化操作的系统。本文将通过 Qt Creator 来实现其功能,Qt Creator 能够提供一个便捷的 GUI 应用程序开发框架。使用 Qt 能够使其实现对图结构的可视化,通过 Qt 自身包含的绘图功能和组件则可以用于处理图算法的核心逻辑。软件的功能主要包括其图结构的展示、经典图算法的逐步演示以及随机图生成等,本文将从背景、开发工具、GUI 设计、图算法可视化、随机图生成以及对比实验等多方面讲述对于图算法可视化的实现。

**关键词**:Qt Creator;图算法;可视化;随机图

## Abstract

With the popularization of computers in daily life, people begin to have a growing demand for computer learning. The learning of computer-related information is inseparable from the learning of graph algorithms, which are widely applied in network analysis, routing, social network mining and other fields. Graph theory is an important branch of computer science, which mainly focuses on how to organize vertices and edges and design algorithms upon them. In order to better understand its basic content and its operating principle, design and development of a simple and basic graph algorithm visualization operation system. This article uses Qt Creator, which provides a convenient GUI application development framework. Using Qt enables us to realize the visualization of graphs, and the drawing facilities and components contained by Qt itself can be used to process the core logic of graph algorithms. The function of software mainly includes the display of graph structures, the step-by-step demonstration of classical graph algorithms and the generation of random graphs. This paper will describe the realization of graph algorithm visualization from the background, development tools, GUI design, graph algorithm visualization, random graph generation and comparative experiments.

**Keywords**: Qt Creator; Graph Algorithm; Visualization; Random Graph

# 第 1 章 绪论

## 1.1 研究背景及意义

图论与图算法是计算机学科的核心课程内容之一,对培养学生的算法思维、深入理解和掌握信息技术学科知识和实践方法、形成学科核心素养,具有十分重要的作用[1]。图算法是数据结构与算法中的重要组成部分,它通过研究顶点与边所构成的离散结构,来描述对象与对象之间的关系,在网络分析、路径规划、社交网络挖掘等领域有着广泛的应用,如何能够快速了解其图算法的核心内容,方便能够更快的了解其知识,而不单单是简单的从书本上学习,通过设计一套完整有效的图算法可视化系统,使其结构和算法能够更加清晰易懂,并在其基础上能够解决一些实际问题,其对于计算机的认识能够加深、对于其他算法内容的学习更是能够事半功倍。

图论与图算法作为信息技术学科的专业基础内容,是对于计算机考研的必考内容之一,也同样是其他计算机课程的基础[2]。图算法课程的教学内容相对较多,知识点相对分散,目前课程的教学活动都是按照图的存储结构、遍历算法、最短路径、最小生成树、拓扑排序的顺序进行的,通过可视化的方式可以更直观的了解其内容[2]。

图算法可视化系统采用了可视化的图结构与用户良好和灵活的界面[3]。实现图算法可视化的意义包含对教学学习的帮助,通过可视化呈现图结构与算法步骤,可以帮助初学者更好地理解和掌握图论的概念和算法实现,同时,对于图算法的可视化可以使学习过程更加生动有趣,从而提高学习效率;其还包含对算法研究的分析,图算法可视化可以帮助研究人员更好地理解和分析算法,而算法是学生进行学习的一个主要内容,算法更是在图论中重要的部分之一,其算法更是能够使程序更加便利,良好的算法不仅可以节约时间,还能提高系统的稳定性[4]。还可以更快的发现算法的优点和不足之处,并对算法进行改进,此外,通过可视化技术,研究人员可以更加直观的展示算法的执行过程和结果,更容易得到有关算法行为的洞察;其中还可以辅助软件开发,在软件开发的过程中,图算法是一个非常重要的组成部分。使用图算法可视化可以帮助开发人员更好地理解和调试代码中涉及的图结构,提高代码质量和效率。

## 1.2 国内外研究现状

早期的计算机主要应用于科学计算,随着计算机的发展和应用范围的拓宽,其需要解决的问题越来越多,人们开始对计算机处理和研究的对象进行探究,即研究数据的特征之间的联系,并通过归类存储实现对数据的存储,从而提高计算机对数据处理的效率。

图论是计算机科学中研究顶点与边的组织、存储和算法的一门学科。它关注于如何用适当的方式来组织图数据,以便于程序的设计和实现。具体而言,图算法包括了遍历算法、最短路径、最小生成树、拓扑排序、网络流等多种形式,并提供了对这些结构进行操作的算法。对于不同的应用场景,需要选择合适的图模型和算法,以达到最优的效果。基础的图算法有 DFS、BFS、Dijkstra、Prim、Kruskal、Kahn 拓扑排序等。

国内在图算法可视化教学上也有较为广泛的研究和应用,在高校中,图论与算法是计算机科学与技术等相关专业必修课之一,许多高校在该领域拥有着雄厚的师资力量和实验室设施,同时也有丰富的研究成果,总体来说,国内在图算法领域的研究和教学应用已相当成熟。

国外对"图论与算法"教改主要是围绕计算机辅助教学工具的开发和改进展开的。在过去的 20 多年里,国外大学已开发出了一些辅助该门课程教学的可视化工具(visualization tools)和其他教学辅助工具[5]。对此国外还有大学专门研究深入了解学生对算法和图论其核心概念的学习程度[6]。

图论与算法是计算机科学与技术专业及相关专业的一门专业基础课程[7],在计算机中有着重要地位。其课程不仅涉及计算机硬件知识,比如编码理论、存取方法,而且还与软件研究关系密切[8],其主要目的在于培养学习者的图抽象能力以及学会用计算机思维思考问题的能力[9]。图论中常见的逻辑结构有顶点、边、路径、环、连通分量等,这些结构比较抽象,学生理解起来有难度,更是遇到实际问题时不能够准确确定组织数据的逻辑结构[10]。

## 1.3 可行性分析

在学习计算机的过程中,图论与图算法是必不可少的内容,算法与图结构更是重中之重,对图算法进行可视化的原理是通过对可视化的算法对输入的图数据进行处理和分析,然后在算法的执行前、算法在执行过程中和算法执行完毕后这三个阶段中根据对图结构的变化进行图形化显示,以供学生观察、分析和理解算法[11]。该系统通过 Qt 对图算法的可视化操作。

Qt 的设计界面部件种类多样,初学者可以通过直接拖拽控件进行软件界面的布局,对于控件的坐标、大小等各种属性的调整也直观方便。Qt 信号槽机制也使用户与系统的交互显得更加便捷,其信号槽是一种在 Qt 框架中使用的通信机制,允许对象之间进行异步通信,而不必显式地连接彼此。一个信号是一个对象发出的事件,而槽则是接受该事件的对象上的函数,通过将信号与槽连接起来,可以实现当某个事件发生时自动调用特定的函数,从而使得对象之间的交互更加灵活和高效。例如,当用户单击按钮时,会发出"clicked"信号,可以将该信号连接到处理单击事件的函数上,从而触发相应操作。

Qt 提供了丰富组件与工具,Qt 作为一款优秀的 GUI 框架,拥有很多用于构建图形界面的控件和工具类,例如,QWidget、QGraphicsView、QGraphicsScene、QMainWindow 等等,这些组件可以轻松创建出各种自定义的图形元素和交互式控件。开发者可以根据对系统的需求,选择合适的组件进行布局管理,此外,Qt 还提供了 Qt Designer 工具,允许通过拖拉的方式设计界面,十分方便快捷;Qt 还具有强大的图形渲染引擎,可以高效的绘制各种图形元素,并且支持透明度、边框、阴影、渐变等自定义外观的设置,同时,Qt 还提供了一系列的动画框架,可以方便的实现各种动画效果,对系统的开发提供了强大的技术支持;Qt 具有良好的跨平台特性,可以在 Windows、Linux、macOS 等操作系统上运行,并且也支持移动端开发,意味着基于 Qt 开发一次,能够在多个平台上运行;Qt 采用信号槽机制来处理事件,使得开发者可以更加灵活的处理用户与界面之间的交互,当用户与界面元素交互时,Qt 会自动调用与之相关联的槽函数,从而实现事件的处理,方便实现系统中的添加、删除等操作;Qt 作为一款广泛使用的开源框架,拥有庞大的社区和完善的官方文档,在使用 Qt 进行系统开发时,可以从社区或者官方文档中获取丰富的学习资源。

Qt 提供了丰富的图形界面开发工具和库,可以轻松地创建各种自定义的图形元素和交互式控件。同时,Qt 还提供了强大的图形渲染引擎和动画框架,可以实现高效的图算法可视化效果。此外,Qt 具有跨平台性和信号槽机制等优秀特性,使得图算法可视化的开发变得更加便捷和灵活。

# 第 2 章 开发工具和环境配置

## 2.1 系统介绍

图算法可视化的实现是通过 Qt 实现,其系统是一种用于图形化展示各种图算法操作的软件工具,用于对图算法的展示。由两个部分组成,其一是图算法可视化界面的设计,另一是编程环境的构造,通过编写代码,有可视化界面展示其运行结构。

本文所设计的系统通过 Qt Creator 实现,通过 Qt Creator 的 GUI 交互界面来实现用户操作,并通过其 Qt 所自带的绘图功能实现对图结构的可视化,并通过 Qt 自带的 Graphics View 框架来实现图节点与边的显示。软件主要包含向用户展示其图结构的逻辑形态,并通过对算法进行函数设计,通过控件调用来实现函数调用,实现算法的可视化,系统的设计包含图的构造、随机图的生成、经典算法的演示等其逻辑结构内容,并能够实现其算法的核心内容,实现对图的编辑,包含手动添加节点和删除节点的控制;对随机图的实现,实现其多种模型的生成功能,并实现对算法运行过程的高亮显示效果,使其能够更加方便的展示。

## 2.2 系统开发工具介绍

Qt 是标准框架的图形应用程序,它使用"一次编写,随处编译"的方式为开发跨平台的应用程序提供了一个标准的 C++ 图形应用程序开发框架[12]。

Qt 拥有更加直观和强大的 API,其作用于各种平台的软件开发[13],提供 GUI、网络、数据库、XML 和多线程等支持。本次所用到的 Qt Creator 是一款 Qt 推出的轻量级跨平台集成开发环境[14]。Qt Creator 提供了支持本地和远程调试,可以在运行时检查变量的值、观察函数调用栈、设置断点等,同时支持多种构建工具(如 qmake 和 cmake),可以生成可执行文件和库文件,并支持部署到不同平台,同时集成常见的版本控制系统(如 git 和 subversion),方便管理项目代码,它提供了许多有用的功能,如代码自动补全、语法高亮、调试器和可视化编辑器等[15]。Qt 因为其自身的封装机制处于行业中的领先地位,因此模块化程度也非常的高,为用户开发提供了便利,Qt Creator 支持插件拓展,用户可以根据自己的需求安装不同的插件,从而扩展其功能。在国内,Qt 在安防监控领域、航空领域、航海领域、工业控制领域,都占有比较大的比例。Qt Creator 可以在多个操作系统上运行,包括 Windows、Linux 和 MacOS 等[16]。并且作为一个跨平台框架软件,具有能够与应用程序进行对接的接口,方便对应用进行编码调试[17]。赋予应用开发者人员完成应用程序建立适当且高效的图形化界面程序和后台执行的大部分功能,并提供可扩展性能的面向对象的程序设计模式和以组件为基础的程序设计模式[18]。

本文还使用了 CMake 作为项目的构建工具,通过编写 CMakeLists.txt 来管理 Qt 项目的依赖与编译选项,通过 MSVC 2022 编译器进行 Release 模式的编译,以保证程序运行的效率与稳定性。

## 3.2 界面优化

对于 Qt 的使用运行,可以通过使用布局管理器来实现界面元素的位置和大小,确保窗口大小发生变化时仍然保持相对位置。而对于图算法可视化的运行操作,其必然需要利用 Qt 的绘图功能,其对于绘图功能的使用如果直接重写 paintEvent 函数,在其中绘制过多功能,则会影响其 UI 的响应速度,从而减少其运作效率,通过对其功能进行分类,采用 Qt 的 Graphics View 框架来代替直接重写 paintEvent,在每一个需要执行的功能通过创建不同的 cpp 文件,避免过多的嵌套和复杂的设计,在其需要的功能上通过 QGraphicsItem 派生类的 paint 函数来实现局部绘制,以执行其在运行哪个图元时再调用其绘图功能,从而减缓其运作负荷。

# 第 5 章 随机图生成模块的设计与实现

## 5.1 随机图模型概述

随机图是图论中的一个重要分支,通过引入概率机制生成图结构,用于研究在不确定连接关系下网络的统计特性,在复杂网络、社交网络分析、通信网络仿真等领域有着重要应用。常见的随机图模型包括 Erdős–Rényi 模型、随机几何图模型、无标度网络模型、小世界网络模型等,不同模型反映了不同的网络生成机制。

本系统主要实现了 Erdős–Rényi 模型与随机几何图(RGG)模型这两类经典模型,通过对这两类模型的实现与对比,展示其在边数、平均度、聚类系数、连通性等结构特征上的差异。Erdős–Rényi 模型由 Erdős 与 Rényi 于 1959 年提出[1],是最基本的随机图模型,其特点是边的存在与否相互独立,适合作为各类网络分析的基准模型。随机几何图模型由 Penrose 系统研究[2],其特点是顶点带有空间位置,边的存在与否取决于空间距离,适合用于刻画无线传感网络、地理邻接网络等空间约束的场景。

通过对这两类模型的实现,本系统能够为图算法的运行提供多样化的测试用例,并通过对比实验来研究不同模型对算法性能的影响。

## 5.2 ER 生成机制

Erdős–Rényi 模型有两种等价的表述,其一是 G(n, m) 模型,即在 n 个顶点上随机选取 m 条边;其二是 G(n, p) 模型,即在 n 个顶点上,每对顶点以概率 p 独立连边。通过对这两种模型的实现,可以为用户提供按边数与按概率两种参数化方式。

## 5.3 随机几何图(RGG)的生成机制

随机几何图(Random Geometric Graph,RGG)是一种基于空间几何邻近性的随机图模型,通过在单位空间内均匀撒点并以欧氏距离阈值连边来生成图。其结构定义为:n 个顶点独立均匀地撒在单位正方形 [0, 1]² 内,任意两点欧氏距离不超过给定阈值 r 时,在它们之间连一条边[2]。RGG 模型由于其连接受空间约束,邻居的邻居很可能也在附近,因此具有较高的聚类系数,适合用于刻画无线传感网络、地理邻接系统等具有空间约束的网络场景。

Penrose 在其专著 Random Geometric Graphs 中证明,RGG 的连通性临界半径约为 √(ln n / (π·n))[2],当 r 大于该阈值时,图几乎必然连通;当 r 小于该阈值时,则会出现大量孤立点与小连通分量。这一性质使 RGG 成为研究空间网络相变现象的标准模型。

通过对 RGG 模型的实现,其核心是分两步进行:第一步是在单位正方形内均匀撒点,通过 QRandomGenerator 的 generateDouble 函数为每个顶点生成 (x, y) 坐标;第二步是对所有顶点对计算欧氏距离,如果距离不超过 r 则连边,其核心代码为图 5-5 所示。

图 5-5 RGG 模型生成核心代码

# 第 6 章 不同随机图模型上的算法运行实验

## 6.1 实验环境配置与评估指标

为了能够对不同随机图模型上算法的运行情况进行对比,本章设计了一组对比实验。所有实验在以下环境中完成:CPU 为 Intel Core 系列,内存 16 GB,操作系统为 Windows 11,编译器为 MSVC 2022 / Release 模式,Qt 版本为 6.10。统计脚本基于 Python 3.11,通过对系统生成的图进行二次分析,以保证数据的客观性。

为了能够对图的结构特征进行评估,本实验主要采用以下评估指标:边数,即图中边的总数,反映图的整体密度;平均度,即所有顶点度的平均值,等于 2 · 边数 / 顶点数;最大度,即图中度数最大的顶点的度,反映是否存在"枢纽节点";孤立点数,即度为 0 的顶点数,反映图的连通性;连通分量数,即图中互不连通的子图数量;最大连通分量大小,即最大的连通子图所包含的顶点数;聚类系数,即对所有顶点局部聚类系数的平均,反映"邻居互为邻居"的程度;平均路径长度,即所有可达顶点对之间最短路径长度的平均,反映网络的紧凑程度。

通过对上述指标的统计,可以从多个维度对不同模型的结构差异进行量化对比,为后续的分析提供数据支撑。

## 6.2 基于 ER 模型的实验与分析

通过对 ER 模型在 n = 100、p = 0.10 下的生成实验,运行 20 次取均值,其统计结果如下:边数为 488.4,平均度为 9.77,最大度为 17.8,孤立点数为 0.1,连通分量数为 1.1,最大连通分量大小为 100.0,聚类系数为 0.0967,平均路径长度为 2.25。

通过对实测结果的分析,实测平均度 9.77 与理论期望 p(n-1) = 9.9 高度接近,实测边数 488.4 与理论期望 p · n(n-1)/2 = 495 也基本吻合,说明 G(n, p) 实现是正确的。在连通性方面,由于 p = 0.10 远高于理论连通性阈值 ln(n)/n ≈ 0.046,因此 ER 图在该参数下几乎必然连通,实测连通分量数仅 1.1,孤立点数仅 0.1,与理论一致。

在聚类系数方面,实测值 0.0967 几乎等于概率 p = 0.10,这是 ER 模型的典型特征:由于边的存在与否相互独立,任意三个顶点之间形成三角形的概率约等于 p,因此聚类系数也约等于 p。这一较低的聚类系数反映了 ER 模型缺乏"局部聚集"的特性,与现实网络中普遍存在的"邻居互为邻居"现象有较大差距。

在平均路径长度方面,实测值 2.25 较短,这反映了 ER 模型由于边的随机分布,顶点之间能够通过较少的中转就到达目标,体现了"小直径"的特性。

通过对 ER 模型实验的分析,验证了 G(n, p) 实现的正确性,同时也展示了 ER 模型作为基准模型在结构特征上的不足,为对比 RGG 模型提供了基础。

## 6.3 基于 RGG 模型的实验与分析

通过对 RGG 模型的连通性相变现象的研究,本实验设计了三组参数:r = 0.05(远低于阈值)、r = 0.121(接近理论阈值 r_c)、r = 0.20(高于阈值),在 n = 100 下各运行 20 次取均值,其统计结果如下表所示。

| 参数 | 边数 | 平均度 | 孤立点数 | 连通分量数 | 最大分量大小 | 聚类系数 |
|------|------|--------|----------|------------|--------------|----------|
| r=0.05 | 36.0 | 0.72 | 49.8 | 69.3 | 5.2 | 0.5829 |
| r=0.121 | 203.9 | 4.08 | 2.3 | 7.8 | 66.3 | 0.6094 |
| r=0.20 | 517.5 | 10.35 | 0.1 | 1.1 | 99.7 | 0.6489 |

通过对三组实验结果的分析,在 r = 0.05 时,由于半径远低于理论阈值 r_c ≈ 0.121,图严重碎片化,出现 49.8 个孤立点与 69.3 个连通分量,最大连通分量仅有 5.2 个顶点,说明此时图基本无法连通;在 r = 0.121 时,处于过渡区,孤立点降至 2.3,连通分量数降至 7.8,最大连通分量上升至 66.3,呈现出"巨型连通分量正在涌现"的相变特征;在 r = 0.20 时,图几乎必然连通,连通分量数仅 1.1,最大连通分量包含 99.7 个顶点,与理论预测高度一致。

通过对 RGG 连通性相变曲线的绘制,通过让 r 从 0.02 逐步增大到 0.30,记录每个 r 值下最大连通分量的相对大小,可以清晰地观察到一条 S 形相变曲线,其曲线如图 6-1 所示。

图 6-1 RGG 连通性相变曲线

在聚类系数方面,RGG 模型在所有三组参数下都保持了 0.58 以上的高聚类系数,这是 RGG 模型最重要的特征:由于连接受空间约束,两个相距很近的顶点的共同邻居很可能也在它们附近,因此构成大量的三角形结构。这一聚类系数远高于同密度 ER 模型的 0.0967,差异接近 6.6 倍,充分体现了几何邻近性对网络结构的强烈影响。

通过对 RGG 实验的分析,实测数据与 Penrose 在其专著中给出的理论预测高度一致,验证了 RGG 生成算法的正确性,同时也展示了 RGG 模型在聚类与连通性上的独特特征,为后续的对比讨论提供了量化依据。

## 6.4 两种随机图模型下算法性能对比总结

通过对 ER 模型与 RGG 模型在 n = 100 下大致等密度参数下的对比实验,其结果汇总为表 6-1 所示。为了使两个模型的密度大致相当,ER 取 p = 0.10,RGG 取 r = 0.18,使两者的边数都在 400~500 之间。

| 模型 | 参数 | 边数 | 平均度 | 最大度 | 聚类系数 | 平均路径长度 |
|------|------|------|--------|--------|----------|--------------|
| ER G(n,p) | p=0.10 | 488.4 | 9.77 | 17.8 | 0.0967 | 2.25 |
| RGG | r=0.18 | 425.9 | 8.52 | 16.9 | 0.6393 | 4.33 |

通过对两个模型的对比分析,可以得出以下几点结论。

第一,在边数与平均度上,两个模型差异不大,均在同一量级,这为公平对比提供了基础;在最大度上,两个模型也都在 17 上下,说明在该参数下两个模型都没有出现明显的"枢纽节点"。

第二,在聚类系数上,RGG 的 0.6393 是 ER 的 0.0967 的约 6.6 倍,这是两个模型最显著的差异。RGG 的高聚类源于几何邻近性:两个相距很近的顶点的共同邻居很可能也在它们附近,因此形成大量三角形;而 ER 的低聚类源于边的独立性:任意三个顶点形成三角形的概率仅为 p^3 ≈ 0.001,因此聚类系数极低。这一差异说明,在需要刻画"局部聚集"特征的应用场景(如社交网络、生态接触网络)中,RGG 模型比 ER 模型更具表达力。

第三,在平均路径长度上,RGG 的 4.33 几乎是 ER 的 2.25 的两倍,这反映了 RGG 模型的另一个重要特征:由于缺少长程连接,信息只能沿空间相邻关系逐跳传递,因此整体直径较大;而 ER 模型由于边的随机分布,顶点之间能够通过较少的中转就到达目标,呈现"小直径"的特征。

通过对两个模型的对比分析,可以看出 ER 模型适合作为"无结构基准"用于理论分析,而 RGG 模型则更适合用于刻画具有空间约束的真实网络。在 ER 模型上运行 BFS、Dijkstra 等遍历类算法时,由于路径较短,算法的搜索深度也较浅,运行步数较少;在 RGG 模型上运行同类算法时,由于路径较长,算法的搜索深度增加,运行步数也相应增加。这一对比对于理解算法在不同网络结构上的实际性能具有重要意义。

通过对两种随机图模型下算法运行实验的设计与分析,本系统不仅实现了对图算法的可视化,更通过量化的对比实验展示了不同模型在结构特征上的本质差异,为图论与复杂网络的教学提供了一份可参考的实验素材。

# 第 7 章 总结与展望

## 7.1 总结

通过 Qt Creator 能够实现部分图算法的可视化和随机图的生成,在通过 Qt 自带的绘制功能能够使其能够可视化。其中的主要功能主要包括实现图论中的常用算法,其生成的界面中包含图的编辑、随机图的生成、算法的单步演示等可视化操作,其中算法主要是通过 Graphics View 框架来生成,对于点击不同的按钮实现在一个窗口进行功能切换,在算法的事件中,通过对每一种算法继承 AlgorithmBase 抽象基类并实现其 step 函数,以实现对算法的单步执行;在 Dijkstra 的事件中,通过优先队列来维护当前最短距离;在 Prim 与 Kruskal 的事件中,通过对边的高亮来展示生成树的构造过程,Prim 通过优先队列实现,Kruskal 通过并查集实现;在 Kahn 拓扑排序的事件中,通过入度数组与队列来实现对有向无环图的排序。在通过对 ER 与 RGG 两类随机图模型的实现,使用户能够生成丰富多样的图结构用于算法的测试与研究,在通过对两种模型的对比实验,展示了 ER 模型与 RGG 模型在边数、平均度、聚类系数、平均路径长度等方面的本质差异,为图论与复杂网络的教学提供了一份可参考的实验素材。

## 7.2 不足与展望

由于知识与能力的限制,对于图论与图算法的了解也不是完全深刻,只能够对其部分经典算法进行可视化,无法完全覆盖图论中的所有算法,如最大流、强连通分量、最大匹配等高级算法尚未实现;在其对于界面交互的修改过程中也遇到了一些 BUG,例如在拖拽顶点时偶尔会出现连线的轻微错位,在反复测试后基本稳定,但在大规模图(n 大于 500)下仍存在一定的性能问题,对此感到沮丧;并且在对鼠标事件的学习中,对于其做出的触发效果也感到不足,而其中对于 RGG 模型的实现更是通过多种方式尝试才达到了较为理想的效果,达不到一开始的预计;而一开始对于有向图复杂算法的设计,也由于时间的紧迫而未能完成,对此感到沮丧;而其中对于 Qt 的使用也是无法完全利用,对于布局算法与可视化效果的美化也近乎没有,对于 Qt 强大的 GUI 交互工具,也不能完全使用,其做出的系统功能也有一些 BUG,在进行调试的时候花费了大量的时间,导致后续的对比实验也十分忙碌,且收获相对有限,并且对于代码的编写过程,也同样有许多问题,其中过多的重复定义使系统未能实现一开始设想的简洁。通过这次的学习,对于 Qt 的学习也是更加深刻了,对于其中的逻辑也有一定的了解了,而对于现在的系统,在后续也会进行更加完善,对于有向图算法、动态图、大规模图的网格桶加速、力导向布局等的实现也会在后续进行尝试,并对系统美化也会进行学习,为实现其目标,本人将会对 Qt 进行更加深化的学习,争取能够更加完善其系统功能。
