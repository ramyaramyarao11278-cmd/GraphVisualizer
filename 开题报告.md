## 一、选题的目的、意义及国内外研究现状

### 1.1 选题目的与意义

图论与图算法是计算机科学与技术专业的核心课程内容,在网络分析、路径规划、社交网络挖掘、生物信息学、人工智能等领域有着广泛的应用。然而,与一维的数组、链表等线性结构不同,图缺乏天然的几何形态,同一张图在平面上可以有多种合法的绘制方式;与此同时,图算法的执行过程往往涉及大量节点与边的状态变化,例如 Dijkstra 算法中节点距离的逐步松弛、最小生成树算法中边集的渐进构造、拓扑排序中入度的逐步消减等。这些动态过程如果仅依靠静态的伪代码或文字描述,初学者很难形成准确的心智模型,常常出现"看得懂代码却讲不清原理"的现象。

为了帮助学习者更直观地理解图算法的核心思想,通过设计与实现一款基于 Qt 的图算法可视化系统,能够将抽象的算法过程以图形化的方式展示出来,使学习者能够通过单步执行、暂停、重置等交互方式,观察算法在每一步的状态变化,从而加深对算法的理解。该系统不仅可以作为图论教学的辅助工具,也可以作为复杂网络研究中随机图模型对比的实验平台,具有较为广泛的应用价值。

通过本课题的研究,能够加深对 Qt 框架、C++ 程序设计、面向对象设计模式的理解,同时也能够通过对随机图模型的研究,加深对复杂网络理论的认识,为后续的学习与研究奠定基础。

### 1.2 国内外研究现状

国外早期的代表性算法可视化工作可以追溯到 20 世纪 80 年代,Brown 与 Sedgewick 开发的 BALSA 系统、Stasko 等人开发的 TANGO 与 XTANGO 系统奠定了算法动画的范式基础。21 世纪以来,随着 Web 技术的普及,浏览器端的可视化工具迅速涌现,其中代表性的有新加坡国立大学开发的 VisuAlgo,该系统覆盖了从基础数据结构到高级图算法的近 30 个主题,已成为应用最广泛的在线工具;以及开源社区维护的 Algorithm Visualizer,该工具支持多语言代码同步执行。这些 Web 工具的共同优点是免安装、易传播,但其局限性也比较明显,即图规模受限且难以集成系统化的实验对比功能。

桌面端可视化工具方面,Gephi 是复杂网络分析的事实标准,功能强大但定位偏向"科研分析"而非"教学演示",对算法的逐步执行过程支持较弱;Cytoscape 主攻生物网络,同样不以算法教学为核心目标。

国内方面,部分高校在数据结构课程中开发了配套的可视化课件,但多以 Flash 或独立 GUI 形式存在,长期维护与跨平台兼容性较差。整体来看,"面向教学、覆盖多种随机图模型、支持算法步进、跨平台、开源"这五个特性同时满足的工具仍然较为稀缺,这构成了本课题的设计动机。

在随机图模型方面,Erdős 与 Rényi 于 1959 年提出的 ER 随机图是最经典的随机图模型;Penrose 在 2003 年的专著中系统研究了随机几何图(RGG)模型;Watts 与 Strogatz 于 1998 年提出了小世界网络模型,Barabási 与 Albert 于 1999 年提出了无标度网络模型,这些模型为复杂网络研究奠定了理论基础。本课题将在 Qt 平台上实现 ER 与 RGG 两类经典随机图模型,并通过对比实验展示它们在结构特征上的差异。

## 二、课题任务、重点研究内容、研究途径

### 2.1 课题任务

本课题的总体任务是设计并实现一款基于 Qt 6 / C++17 的图算法可视化桌面系统。系统需要满足以下三方面要求:

第一,在图编辑功能上,能够支持手动创建顶点与边,支持有向图与无向图的切换,支持加权图与无权图,提供 JSON 格式的图保存与加载功能,使用户可以将构造好的图持久化保存。

第二,在随机图生成功能上,能够生成 Erdős–Rényi(包括 G(n,p) 与 G(n,m) 两种变体)与随机几何图两类经典随机图模型,生成参数可由用户自由调节,并提供实时的参数预估提示,使用户在生成之前对结果有所预判。

第三,在图算法可视化功能上,能够实现深度优先搜索、广度优先搜索、Dijkstra 最短路径、Prim 与 Kruskal 最小生成树、Kahn 拓扑排序等经典图算法的单步可视化演示,提供"开始 / 单步 / 暂停 / 重置"的播放控制,并通过颜色高亮反映算法在每一步对节点与边的访问状态。

### 2.2 重点研究内容

本课题的重点研究内容包括以下几个方面。

**图数据结构的设计**:研究采用何种存储结构(邻接矩阵、邻接表)来表示图,使其在空间效率与访问效率之间取得平衡;研究如何使用 QMap 等 Qt 容器来实现顶点与边的快速查找;研究如何通过 Qt 的信号槽机制实现数据层与视图层的自动同步。

**算法可视化的统一抽象**:研究如何设计一个抽象基类,使各类图算法都可以共享统一的"开始 / 单步 / 暂停 / 重置"控制接口,从而降低视图层与算法层的耦合度,提高系统的可扩展性。

**随机图模型的实现与验证**:研究 ER 与 RGG 两类模型的生成算法,对 RGG 模型重点研究其连通性临界半径的理论性质,并通过独立的验证脚本对生成结果进行结构指标的统计分析,确保算法的正确性。

**GUI 界面的设计与优化**:研究基于 Qt Graphics View 框架的图绘制方式,使顶点与边在用户拖拽时能够实时更新位置;研究通过 QSS 样式表对界面进行美化,使界面达到现代化的视觉效果。

### 2.3 研究途径与技术路线

本课题采用"理论调研→架构设计→分模块实现→集成测试→实验对比"的研究路线。

在理论调研阶段,通过查阅图论、随机图、复杂网络的经典文献,梳理本课题涉及的核心理论;通过查阅 Qt 官方文档与已有的 Qt 项目,了解 Qt 6 的最新特性与开发模式。

在架构设计阶段,通过 UML 类图与模块依赖图,明确系统的分层结构,确定各类的职责与接口。

在分模块实现阶段,按照"数据层 → 算法层 → 生成层 → 视图层 → 控制层"的顺序逐层实现,每完成一层都进行单元测试,确保模块的正确性。

在集成测试阶段,将各层组装为完整的系统,通过手动测试与自动化脚本验证系统的整体功能。

在实验对比阶段,通过 Python 验证脚本对 ER 与 RGG 两类模型在 n = 100 的规模下进行结构指标的对比,生成对比表与相变曲线图,作为论文的实验数据。
